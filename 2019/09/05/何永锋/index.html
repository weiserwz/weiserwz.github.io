什么是hexo

度娘谷歌搜索。。。。。。。
度娘搜索node.js
由于换电脑，hexo需要移植，恰巧之前的域名到期了。不知道为什么狗爹没给我来邮件提示域名过期，超过一个月导致没法续费了。于是换回了hexo的默认域名。在折腾中遇到了几个新的问题。

一、DeprecationWarning: fs.SyncWriteStream is deprecated
问题：
node和hexo插件的版本带来的问题：在node8.x的版本中，fs.SyncWriteStream被弃用了。

解决：
更新如下插件：

npm install hexo-fs –save
npm install hexo-deployer-git@0.3.1 –save
npm install hexo-renderer-ejs@0.3.1 –save
npm install hexo-server@0.2.2 –save
1
2
3
4
二、域名改为默认
1、删除source/CNAME文件。
2、修改hexo目录下_config.yml（不是theme下）的url ： http://liuyongshun.github.io
3、换个浏览器或清除缓存打开。

三、文件大小写导致hexo的404
1.git提交到github默认是不区分大小写的。
解决： 找到.deploy_git/.git/config打开，修改 ignorecase 为false 。ignorecase = false

2.github上原来有了小写的文件夹，又提交了大写的文件，想要删除小写的。
解决：

–cached会保留本地的不被删除。在.git 同级目录下启动git。执行如下命令（这会不用hexo的命令）。
1.git rm -r –cached cc/dd/zz
2.git commit -m “”
3.git push （可不加：origin master）
1
2
3
4
关于这里可以延伸解决另一个问题，就是有些时候你提交了没有的文件夹到github，例如.idea。即使后来你增加.gitignore,github上也会存在.idea。而当你想删除github上的.idea但不删除本地的.idea则同上操错
